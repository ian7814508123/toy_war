# 玩具戰爭網頁復現規格整理

## 1. 專案定位

本專案的目標不是單純重畫畫面，而是復現一款「以瀏覽器為主要平台」的 2D 基地經營 + 進攻掠奪遊戲。

從目前整理出的資料來看，遊戲核心由以下三層組成：

1. 基地經營：土地、建築、資源生產、倉儲容量、升級。
2. 軍事養成：士兵工廠、訓練中心、研究中心、英雄大廳。
3. 攻防互動：運兵、攻城車、敵方基地破壞、建築受損與修復。

這種結構非常適合先做成 2D 網頁遊戲原型，再逐步擴成完整版本。

## 2. 建議工具選型

### 主方案

- 遊戲引擎：Phaser 3
- 語言：TypeScript
- 建置工具：Vite
- 地圖編輯：Tiled
- 像素美術與動畫：Aseprite

### 為什麼適合這款遊戲

- Phaser 3 對 2D 場景、UI、滑鼠互動、精靈動畫、Web 部署都很直接，最接近舊式網頁遊戲的開發節奏。
- TypeScript 很適合管理大量建築、兵種、資源與升級規則，能降低後期資料混亂。
- Vite 啟動快，適合反覆測試 UI 與遊戲流程。
- Tiled 適合編輯基地格子、障礙、地塊與可擴張區域。
- Aseprite 適合處理建築升級外觀、士兵待機/移動/攻擊動畫。

### 後續擴充工具

- 本機資料格式：JSON
- 後端：Node.js
- 即時多人或同步戰鬥：Colyseus
- UI 純量大時可考慮 HTML UI + Phaser 場景混合

## 3. 目前已整理出的遊戲元素

### 地圖與基地

- 初始基地土地大小為 `1000m x 1000m`
- 外圍土地可逐塊擴張
- 每次擴張單位看起來是 `100 x 100`

### 核心建築

- 主控台
  - 決定其他建築可升級上限
  - 屬於基地核心建築
- 資源工廠
  - 定時生產資源
  - 生產速度與儲量上限受等級影響
  - 需要手動收取
  - 被摧毀後停止生產，修復後恢復
- 油庫
  - 生產高階士兵用資源
  - 石油位於地下，因此不像一般資源那樣有明顯掠奪損失
- 倉庫
  - 控制各資源儲存上限
  - 多個倉庫直接累加總容量
  - 單項資源滿倉時，對應工廠停止生產
  - 被摧毀重建後會有部分資源損失

### 軍事與成長建築

- 一般士兵工廠
  - 消耗塑膠生產一般兵
- 特種士兵工廠
  - 消耗塑膠、石油等資源生產特種兵
- 訓練中心
  - 解鎖或強化士兵能力
- 鐵匠鋪
  - 合成寶石、強化裝備
  - 提升攻擊與防禦體系整體能力
- 英雄大廳
  - 英雄研究與培養
  - 建築等級提升可增加可持有英雄數
- 研究中心
  - 士兵轉生後可學習進階技能
  - 分為普通技能與專屬技能

### 戰鬥與運輸相關

- 運輸中心
  - 進攻模式時將士兵運往敵方基地
- 士兵房屋
  - 生產出的士兵先駐紮於房屋中
  - 可透過升級或增建提高容量
- 攻城車
  - 遠距離攻城武器
  - 會把積木與螺絲等基礎物資投擲到敵方基地以破壞建築

### 已出現的資源

- 塑膠
- 水晶
- 積木
- 螺絲
- 石油
- 鐵或寶石類高階素材目前可視為後續擴充

## 4. 建議的系統拆分

### A. 基地系統

- 地塊資料
- 建築放置規則
- 升級條件與升級時間
- 建築狀態切換：正常、生產中、已滿、受損、修復中

### B. 資源系統

- 被動生產
- 手動收取
- 倉儲上限判定
- 建築受損造成的停工與損失

### C. 單位系統

- 一般兵
- 特種兵
- 英雄
- 攻城單位

### D. 戰鬥系統

- 進攻編隊
- 目標選擇
- 攻擊距離
- 建築耐久
- 受損與摧毀
- 戰後結算

### E. 成長系統

- 訓練中心升級
- 士兵轉生
- 研究技能
- 裝備與寶石強化

## 5. 最小可玩版本 MVP

第一版不要一次把全部系統做完，先做能證明玩法成立的版本。

### MVP 必做

- 基地地圖與格子放置
- 主控台
- 2 種基礎資源工廠
- 倉庫
- 一般士兵工廠
- 1 種士兵
- 簡化版運輸/出征
- 敵方基地 1 種防守建築
- 建築血量、破壞、簡單戰鬥結算

### 第二階段再做

- 特種兵
- 英雄
- 研究中心
- 鐵匠鋪
- 攻城車
- 土地擴張
- 更多建築狀態外觀

## 6. 建議資料結構

### 建築定義

每個建築至少需要：

- `id`
- `name`
- `category`
- `size`
- `maxLevel`
- `buildCost`
- `upgradeCost`
- `upgradeTime`
- `hp`
- `storageCap` 或 `productionRate`
- `stateSprites`

### 單位定義

每個單位至少需要：

- `id`
- `name`
- `type`
- `cost`
- `trainTime`
- `hp`
- `attack`
- `attackRange`
- `moveSpeed`
- `carryCap`
- `skills`

### 資源定義

- `id`
- `name`
- `icon`
- `maxStorageRule`
- `producerBuildings`

## 7. 開發順序建議

1. 先把所有建築、士兵、資源整理成 JSON 資料表。
2. 用 Phaser 做基地主畫面與建築放置。
3. 實作資源生產、收取、倉庫上限。
4. 加入士兵生產與駐紮。
5. 做一個簡化攻擊場景或戰鬥模擬流程。
6. 最後再補研究、英雄、進階兵種與外觀變化。

## 8. 現階段結論

如果目標是「盡量保留當年網頁小遊戲的操作感與部署方式」，最適合的技術路線是：

`Phaser 3 + TypeScript + Vite + Tiled + Aseprite`

這條路線的優勢是：

- 最接近瀏覽器原生遊戲
- 原型開發快
- 2D UI 與數值系統好維護
- 之後要補後端或多人功能也容易接上

不建議一開始直接使用較重的 3D 或大型引擎流程，否則很容易把時間花在部署、包體與相容性，而不是復現玩法本身。
