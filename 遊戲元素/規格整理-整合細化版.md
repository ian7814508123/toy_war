# 玩具戰爭網頁復現整合細化規格

## 1. 整體定位

依照目前整理出的三份文件：

- `遊戲元素.md`
- `士兵&英雄.md`
- `防禦建築.md`

這款遊戲可明確定義為：

`2D 網頁基地經營 + 非同步攻防掠奪 + 編隊出征`

它不是傳統即時 RTS，也不是純塔防，而是偏向：

- 平時經營基地
- 生產資源與兵力
- 配置防禦建築
- 組建出征部隊
- 攻擊他人基地並破壞建築、掠奪資源

因此在復現時，最適合先做成「基地主畫面 + 戰鬥模擬場景」的雙模組結構。

## 2. 建議技術方案

### 主方案

- 引擎：`Phaser 3`
- 語言：`TypeScript`
- 建置：`Vite`
- 地圖編輯：`Tiled`
- 美術切圖與動畫：`Aseprite`
- 本機資料配置：`JSON`

### 為什麼這樣選

- Phaser 很適合 2D 建築放置、UI、滑鼠操作、精靈動畫、碰撞與簡易戰鬥模擬。
- TypeScript 很適合承接大量兵種、塔、防禦、技能、升級與資源規則。
- Vite 適合前期反覆迭代。
- Tiled 可以負責地塊、建築占地、障礙與防線測試。

### 不建議一開始使用的方案

- Unity WebGL：偏重，對瀏覽器小遊戲原型不夠輕。
- Godot Web：可行，但若主目標是瀏覽器體驗，前期整體效率仍不如 Phaser 直接。

## 3. 已整理出的核心玩法循環

### 基地內循環

1. 取得土地並擴張基地。
2. 建造主控台、資源工廠、倉庫、兵工建築。
3. 生產資源並手動收取。
4. 用資源生產士兵、培養英雄、研究技能。
5. 建置防禦塔、碉堡、城牆、地雷與防空塔。

### 攻防循環

1. 選擇出征部隊。
2. 從運輸中心或出征介面發動攻擊。
3. 士兵依目標偏好自動尋路與攻擊。
4. 防禦塔依射程與類型自動反擊。
5. 建築被破壞後觸發停工、失能或資源損失。
6. 戰鬥結束後結算破壞率、掠奪量、英雄法力消耗與存活兵力。

## 4. 基地與建築系統細化

### 4.1 地圖與地塊

- 初始基地土地為 `1000m x 1000m`
- 外圍可擴張，單位看起來是 `100 x 100`
- 建議實作時不要直接使用真實公尺，改為「格子座標」
- 建議換算：
  - `1 格 = 10m`
  - 初始基地為 `100 x 100 格`
  - 每次擴張為 `10 x 10 格`

這樣之後做碰撞、放置、射程和尋路會簡單很多。

### 4.2 主要建築職責

- 主控台
  - 基地核心
  - 決定多數建築的等級上限與解鎖
- 資源工廠
  - 定時生產基礎資源
  - 需要手動收取
  - 滿倉停產
  - 被摧毀時停工
- 油庫
  - 生產高階兵種所需資源
  - 與一般資源的掠奪規則可不同
- 倉庫
  - 提供分資源類型的儲量上限
  - 多個倉庫直接加總
- 一般士兵工廠
  - 生產基礎兵種
- 特種士兵工廠
  - 生產高階兵種與功能兵
- 訓練中心
  - 士兵解鎖與強化
- 研究中心
  - 士兵轉生後進一步學技能
- 英雄大廳
  - 持有、升級與管理英雄
- 鐵匠鋪
  - 寶石與裝備強化
- 士兵房屋 / 碉堡
  - 管理士兵駐紮人口
- 運輸中心
  - 攻擊派兵入口

### 4.3 建築狀態

建議所有建築統一使用以下狀態：

- `idle`：正常待機
- `producing`：生產中
- `full`：產能已滿待收取
- `damaged`：受損但仍存在
- `disabled`：被摧毀或停工
- `rebuilding`：修復中
- `upgrading`：升級中

### 5.1 已確認的資源

- 水晶
- 積木
- 螺絲
- 石油

### 5.2 建議資源分類

- 基礎建築資源
  - 積木
  - 螺絲
  - 水晶
- 單位生產資源
  - 塑膠
  - 石油
- 後期強化資源
  - 寶石
  - 鐵
  - 特殊材料

### 5.3 資源規則

- 工廠會累積未收取資源
- 倉庫滿時，對應資源停止生產
- 工廠被破壞後：
  - 暫停生產
  - 未收取資源會按比例損失
- 倉庫被破壞後：
  - 儲存資源按比例損失
- 蜘蛛兵對資源建築有額外掠奪加成

依照你整理出的描述，兵種其實已經很清楚地分成幾個戰術角色。

## 6.3 建議目標優先規則

這一塊是戰鬥手感的核心，建議先明文化。

- `all_buildings`
  - 最近可達建築
- `defense_first`
  - 防禦塔 > 碉堡 > 其他建築

- `support_heal`

## 7. 英雄系統細化

## 7.1 英雄共通規則

- 英雄不同於普通士兵
- 最多可持有 6 位
- 英雄大廳升級會增加可持有數
- 英雄可透過餵食提升等級
- 英雄有 `法力值`
- 法力值足夠時，裝備技能才會生效
- 每次參與攻擊消耗 `80` 法力
- 每次參與防守消耗 `15` 法力
- 法力值可隨時間自動恢復

這代表英雄本質上是：

`可持續培養的稀有單位 + 帶戰前準備成本的技能載體`

## 7.2 英雄定位建議

| 名稱 | 定位 | 核心特色 |
| --- | --- | --- |
| 金剛 | 主坦英雄 | 高血、高防、中攻 |
| 黑騎士 | 刺客英雄 | 高攻、高速、低血 |
| 小飛俠 | 飛行 Buff 英雄 | 提升周圍友軍速度與防禦 |
| 鐵皮 | 反防禦英雄 | 優先打防禦建築，減免地雷傷害 |
| 天行號 | 連擊法術英雄 | 連續攻擊最多三次 |
| 探測器 | 偵查 / 輔助英雄 | 目前資料不足，先保留 |

## 7.3 英雄技能系統建議

建議第一版不要做成太複雜的主動施法，而是先做：

- 被動技能
- 條件觸發技能
- 戰鬥開始自動生效技能

例如：

- 金剛：降低所受傷害
- 黑騎士：首擊加成
- 小飛俠：範圍 Buff
- 鐵皮：對防禦建築增傷、減免地雷傷害
- 天行號：連續三段攻擊
- 探測器：提高索敵視野或提前顯示陷阱

## 9. 戰鬥規則細化

## 9.1 戰鬥場景建議

建議攻擊場景不要做成完全自由操作，而是：

- 玩家在戰前選兵
- 戰鬥中以自動尋路、自動攻擊為主
- 玩家只負責下兵位置或波次

這樣會更接近原本網頁遊戲的實作成本與手感。

## 9.2 尋路規則

- 地面單位不可穿牆
- 飛行單位可忽略多數地面阻擋
- 破牆兵若目標為 `wall_first`，尋路失敗時優先攻擊最近牆體
- 其他單位以最近可達目標為主

## 9.3 傷害規則

建議第一版先用這個簡化公式：

`finalDamage = max(1, attack - armor * armorRate)`

後續再加：

- 暴擊
- 範圍傷害衰減
- 毒傷
- 電擊持續傷害
- 地雷爆炸
- 英雄技能加成

## 9.4 掠奪規則

- 一般建築被破壞時不掉落資源
- 資源工廠被破壞時可掠奪未收取資源的一部分
- 倉庫被破壞時可掠奪儲存資源的一部分
- 蜘蛛額外套用：
  - 資源塔：10%
  - 倉庫：5%

## 12. 目前可直接下結論的設計決策

- 網頁復現最適合使用 `Phaser 3 + TypeScript + Vite`
- 遊戲架構應分成「基地經營」與「攻擊戰鬥」兩個主模組
- 單位 AI 的核心是 `目標偏好`
- 防禦設計的核心是 `射程 + 傷害型態 + 路線控制`
- 英雄設計的核心是 `稀有單位 + 法力限制 + 被動技能`
- 第一版不應一次做完全部兵種與建築，先做一個完整攻防循環最重要

## 13. 仍待確認或補件的地方

- 探測器英雄詳細資料不足
- 天行號缺圖片與完整技能描述
- 部分特種兵只有文字沒有圖像
- 城牆的完整防禦數值未齊
- 碉堡人口容量你已經提出可改版，這需要在真正實作前定案
- 好友擴張、多人互動、社交解鎖是否要保留，建議 MVP 先不做
